## Generative AI Usage

Generative AI tools are used throughout this course as structured learning aids rather than substitutes for understanding. My goal is to use these systems to enhance engagement with networking concepts, validate reasoning, and extend exploration beyond lecture scope.

### Tools Used

- ChatGPT (OpenAI GPT-5.2 via web interface)
- GitHub Copilot (for code-based networking tasks)
- NotebookLM (for lecture-grounded review and summarization)
- Perplexity AI (for standards and RFC-oriented research)

Additional tools may be incorporated as relevant to specific assignments.

---

### How I Use Generative AI

I use generative AI in several distinct ways:

#### 1. Concept Clarification
After lectures, I prompt AI systems to:
- Re-explain mechanisms such as ARP, switching, routing algorithms, and TCP congestion control
- Provide alternate explanations or analogies
- Compare protocols (e.g., Stop-and-Wait vs. Sliding Window)

This helps reinforce understanding and identify gaps in reasoning.

#### 2. Lab Structuring and Debugging
For tool-based labs (e.g., Wireshark, sockets, Mininet), AI is used to:
- Structure lab reports clearly
- Debug environment or Git issues
- Suggest testing methodologies
- Verify terminology and protocol interpretation

All packet-level analysis is performed manually before being formalized into written explanations.

#### 3. Code Assistance (When Applicable)
For assignments involving programming (e.g., socket programming or simulations), GitHub Copilot or ChatGPT may assist with:
- Boilerplate socket setup
- Parsing packet structures
- Implementing routing or congestion control simulations

All generated code is reviewed, tested, and modified to ensure correctness and alignment with lecture material.

#### 4. Standards and Protocol Research
Perplexity AI or similar tools may be used to:
- Locate relevant RFC documents
- Summarize protocol standards
- Clarify IEEE specifications

These summaries are cross-referenced with lecture notes and independent verification.

---

### Study Guide Development Strategy

Throughout the semester, I am building a cumulative study guide that mirrors the structure of the course:

1. Physical Layer  
2. Data Link Layer  
3. Network Layer  
4. Transport Layer  
5. Application Layer  

Generative AI is used to:

- Generate structured outlines aligned with weekly topics  
- Produce practice problems (e.g., subnetting, routing tables, window size calculations)  
- Expand topics beyond lecture scope (e.g., deeper TCP congestion variants, modern Ethernet evolutions, BGP policy mechanisms)  
- Create comparative summaries between related protocols  

The objective is not to replace course material, but to deepen it. AI-generated explanations are treated as drafts that are refined, corrected, and validated against lecture notes and captured network behavior.

---

### Verification and Academic Integrity

All final submissions reflect:

- My own packet captures, lab work, and reasoning
- Direct verification against lecture material
- Manual validation of calculations and protocol behavior

AI outputs are critically evaluated before inclusion. Any inaccuracies are corrected through independent review.

The intent is to use generative AI as an analytical assistant, not an answer generator.
